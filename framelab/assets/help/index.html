    FrameLab Documentation

    Technical documentation for the FrameLab application, including workflow-scoped TIFF intake, hierarchical metadata, calibration-oriented folder structure, session and acquisition tooling, eBUS snapshot workflows, measurement pipelines, analysis plugins, developer workflow, and the bundled offline Help site.

    Use this site by intent: operator workflow, exact reference contract, developer maintenance, handoff context, or cross-cutting troubleshooting.

    User Guide Operator workflows for workflow selection, required folder layout, dataset scan, metadata verification, structure tools, background correction, ROI work, and analysis-plugin interpretation. Reference Canonical file locations, manifest fields, acquisition-mapping keys, eBUS catalog entries, and shortcut contracts. Developer Guide Architecture, state ownership, workflow hierarchy contracts, plugin boundaries, datacard and eBUS integration, UI structure, and packaging rules. Development Repo conventions, change workflow, context strategy, and maintainer troubleshooting for future implementation sessions. Maintenance Tooling map and benchmark workflow for docs, tests, native builds, profiling, and standalone packaging. Handoffs Compact templates for feature, bugfix, and refactor handoffs. Troubleshooting Cross-cutting triage for scope mistakes, scan failures, metadata issues, eBUS anomalies, measurement problems, plugin-loading faults, and offline-help packaging issues.

    How this documentation is organized¶

    This site is divided by question type rather than by audience label alone.

    User Guide¶

    Use the User Guide when the question is:

    • How do I perform this workflow correctly?
    • Which control should I use?
    • What folder structure should I respect?
    • How should I choose a setting, not just click a button?
    • What does this table value, plot, or warning actually mean operationally?

    Recommended entry points:

    • Concepts and Limits
    • Quick Start
    • Workflow Structure and Required Folder Layout
    • Data Workflow
    • Session Manager
    • Datacard Wizard
    • eBUS Config Tools
    • Measure Workflow
    • Analysis Workflow

    Reference¶

    Use Reference when the question is:

    • Where is this file stored?
    • What keys are valid in this JSON file?
    • What does one manifest or catalog field mean exactly?
    • Which keyboard shortcut applies in this context?

    Recommended entry points:

    • Reference Index
    • Config Files
    • Plugin Manifests
    • Acquisition Mapping
    • eBUS Parameter Catalog

    Developer Guide¶

    Use the Developer Guide when the question is:

    • Which module owns this behavior?
    • How does startup, workflow loading, metadata resolution, measurement, or analysis flow actually work?
    • What folder-structure assumptions does the workflow shell make?
    • What contract must be preserved during a refactor or extension?
    • Which assets or scripts must stay synchronized for packaging?

    Recommended entry points:

    • Developer Guide
    • Architecture
    • Plugin System
    • Datacard System
    • eBUS Config Integration
    • UI Structure
    • Packaging

    Development¶

    Use Development when the question is:

    • Which repo conventions should guide this change?
    • What docs should be updated with a code change?
    • How should future sessions use root context docs and local AGENTS.md files?
    • How should maintainer-facing issues be triaged?

    Recommended entry points:

    • Repo Conventions
    • Change Workflow
    • Context Strategy
    • Development Troubleshooting

    For development-session routing, start from the repository-root START_HERE.md.

    Maintenance And Handoffs¶

    Use Maintenance and Handoff Templates when the question is:

    • Which repo-owned helper should run a validation or build?
    • How should benchmark/performance evidence be recorded?
    • What information should be preserved when handing work to another session?

    Recommended entry points:

    • Tooling Map
    • Benchmark Workflow
    • Feature Handoff Template
    • Bugfix Handoff Template
    • Refactor Handoff Template

    Troubleshooting¶

    Use Troubleshooting when the question is:

    • Which subsystem is most likely failing?
    • Is this a workflow-scope problem, dataset problem, metadata problem, eBUS problem, measurement problem, analysis problem, or docs-bundle problem?
    • What should I verify first before changing code or rewriting data?

    Start here:

    • Troubleshooting
    • User Troubleshooting

    Suggested starting points¶

    • First-time operator: Quick Start
    • Required hierarchy and naming: Workflow Structure and Required Folder Layout
    • Dataset and metadata validation: Data Workflow
    • Structure or acquisition preparation: Session Manager
    • Maintenance work: Developer Guide

    Profile maturity note¶

    For most real work, the Calibration workflow profile is the primary documented path. The Trials profile is documented, but it should still be treated as experimental.
